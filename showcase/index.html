Lokt is an AI workout coach for people who want less friction between an idea and an actual training session. Users can ask for a workout in plain English, refine it like a conversation, import a plan from voice or photo, and then log the workout quickly once they start lifting.

The idea

Most workout apps still make planning feel manual.

Existing products often separate advice, generation, tracking, notes, and workout edits into different tools. Lokt tries to pull those pieces together so the user can talk to one coach layer that still results in a real, reviewable workout.

The product bet

More than a tracker. More than a chatbot.

Lokt is meant to feel like texting a coach who can actually do things for you. It should be able to answer naturally, but also create or revise a workout draft when that is what the user wants.

Product Tour

Feature by feature, with the right screen next to it.

Instead of one generic gallery, this section lines each product explanation up with the screen that actually demonstrates it. The goal is to make the product story feel obvious as soon as someone scrolls.

1. Home

Start from one clear product idea.

The home screen is intentionally simple. It tells the user what Lokt is, gives them a strong “Ask Lokt for a Workout” entry point, and still keeps manual building, exercise browsing, and progress review visible.

For a project page, this matters because it shows Lokt is not just an API demo. It is a full product with a clear front door.

  • Start a workout fast
  • Ask Lokt for help immediately
  • Still keep manual and progress flows visible

2. AI Entry

Group the smart tools instead of scattering them.

Once the user taps into Lokt, they can choose how they want to start: typed prompt, voice, photo, or add-on generation. That keeps the AI layer feeling like one coherent system rather than a bunch of disconnected features.

The next screen then lets them type a normal request and get a draft back without forcing them through a long setup flow.

  • Text prompts
  • Voice import
  • Photo import
  • Add-on generation

3. Coach Chat

Give the AI its own real chat surface.

Lokt also has a dedicated coach tab. This is where the app starts to feel different from a normal tracker. Users can talk to the coach naturally, ask training questions, or start from suggested prompts when they do not want to type from scratch.

The quick-prompt composer lowers friction and helps the product feel more like a modern AI experience than a rigid form.

  • Natural questions and follow-ups
  • Suggested prompt starters
  • Fast handoff into workout actions

4. Payoff

The conversation becomes a real workout draft.

This is the core value proposition. Lokt does not stop at advice. It turns the chat into a structured routine draft with a title, exercises, set counts, and a clear save action.

That is the product wedge: something that feels like talking to a coach, but can also do the workout-building work for the user.

  • Real title, exercise order, and set structure
  • Reviewable before saving
  • Advice turns into something actionable

5. Manual Control

AI-first does not mean AI-only.

Lokt still supports a full manual builder for people who want direct control over structure, order, and exercise selection. That makes the product more flexible than a one-shot generator.

It is important that users can move between AI assistance and manual control without feeling like they have to start over.

  • Add and remove exercises manually
  • Adjust set counts and order
  • Keep full control when needed

6. Supporting Surfaces

Library and analytics keep the app useful beyond generation.

The exercise library gives the app a real foundation: search, filtering, and detail access for the movement database. Analytics keeps the long-term loop intact so Lokt is not just about creating workouts, but also about staying with the user over time.

Together, those screens help the product feel complete instead of like a thin AI layer on top of nothing.

  • Browse and filter the exercise database
  • Open exercise details quickly
  • Track history and progress over time

How it works

Lokt is built around one simple user loop.

01

Start with an intent

The user types a prompt, speaks a voice note, uploads a photo, or opens the coach chat to explain what they want.

02

Get a structured draft

The AI layer turns that input into a real workout draft instead of leaving the user with a paragraph of advice they still have to manually rebuild.

03

Refine like a conversation

Users can keep chatting with Lokt to change difficulty, equipment, exercise selection, or overall structure before saving.

04

Log the session quickly

Once the workout begins, Lokt shifts from planning into execution with faster set entry, guided workflow, and cleaner tracking.

How I built it

Native iPhone app, lightweight AI backend, review-first UX.

Lokt is built as a SwiftUI iPhone app with a Node backend for AI generation, revision, photo import, voice import, exercise coaching, and structured draft workflows. I used Codex heavily for product iteration, UI polish, feature implementation, parsing behavior, and the overall coach-chat experience.

Core pieces

  • SwiftUI app for planning, review, logging, and analytics
  • AI backend routes for generation, revision, coach chat, voice, and photo flows
  • Structured JSON drafts so users can review edits before saving
  • Exercise library and metadata system to support matching and swaps
  • Submission-ready showcase site and demo assets built around the real product flow
